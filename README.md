# AI内容洞察工作台

## 一句话介绍

> 粘贴评论，自动分析用户焦虑、生成选题和标题 —— 让内容运营从「拍脑袋」变成「看数据」。

---

## 项目背景

内容运营/新媒体运营每天面对海量用户评论，选题策划长期依赖个人经验和直觉。这个项目试图回答一个问题：**能不能让评论区自己说出该做什么内容？**

传统做法是运营人员逐条翻评论、手动归纳、凭感觉定选题。这个过程耗时且高度依赖个人判断，换一个人可能得出完全不同的结论。

AI内容洞察工作台通过关键词匹配 + DeepSeek大模型两种模式，将评论数据自动转化为可执行的内容策略，覆盖从「数据清洗 → 焦虑分析 → 用户画像 → 内容机会 → 选题/标题」的完整链路。

---

## 项目价值

| 角色 | 价值 |
|------|------|
| **内容运营** | 10 分钟完成原来 2 小时的评论分析工作，直接产出选题和标题 |
| **新媒体运营** | 从评论中识别用户真实焦虑，指导内容选题方向 |
| **内容负责人** | 用数据支撑选题决策，而非依赖个人感觉 |
| **AI 运营** | 展示 AI 在内容工作流中的实际落地场景 |

## 核心功能

### 数据分析
- **评论导入**：批量粘贴或 CSV 文件上传，自动去空行、去重
- **双模式分析**：规则模式（本地关键词匹配）+ AI 模式（DeepSeek 大模型语义分析）
- **高频焦虑识别**：自动归类用户的焦虑类型（AI替代、35岁危机、收入焦虑等）
- **用户画像生成**：基于评论内容推断年龄段、职业、行为特征
- **内容机会分析**：识别用户最关注的问题和最强烈的情绪

### 内容产出
- **10 个选题建议**：可直接进入内容策划的选题方向
- **10 个标题建议**：适合短视频/图文内容的标题
- **选题大纲生成**：点击任一选题，自动生成 5 章节内容大纲

### 辅助功能
- **焦虑占比图表**：饼图/柱状图可视化，支持截图放进报告
- **分析报告导出**：复制文本或下载 Markdown 文件
- **分析历史**：自动保存每次分析，最多 20 条，支持恢复和删除

## 技术栈

| 层 | 技术 | 说明 |
|----|------|------|
| 框架 | React 19 + Vite 6 | 函数组件 + Hooks，ESM 模块 |
| 样式 | Tailwind CSS 3 | 工具类优先，移动端响应式 |
| 图表 | Recharts 3 | 饼图 + 柱状图 |
| 图标 | Lucide React | 统一视觉风格 |
| AI | DeepSeek API (Chat V3) | 可选 AI 分析模式 |
| 存储 | localStorage | 模式/Key/历史持久化 |
| 部署 | GitHub Pages | `/docs` 目录静态托管 |

**纯前端架构** — 无需后端、无需数据库、无需登录。

---

## 项目亮点

1. **产品闭环思维**：从数据清洗到选题产出，覆盖内容运营完整工作流，不只是技术 Demo
2. **降级策略设计**：AI 模式调用失败自动回退规则模式，不中断用户体验
3. **数据安全设计**：API Key 仅存浏览器 localStorage，从不写入文件或上传服务器
4. **渐进式功能迭代**：V1(基础分析) → V2(CSV+AI) → V3(图表+历史+大纲)，每次迭代有明确交付
5. **移动端适配**：完整的响应式布局，支持手机/平板/桌面
